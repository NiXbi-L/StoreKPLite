# Структура React Admin Panel

## Обзор

React приложение для админ-панели MatchWear создано со следующей структурой, где каждая вкладка имеет свою папку с собственными файлами.

## Структура директорий

```
admin_react/
├── src/
│   ├── components/          # Общие компоненты
│   │   └── Layout/          # Layout с сайдбаром
│   │       ├── Layout.tsx
│   │       └── Layout.css
│   │
│   ├── contexts/            # React Context
│   │   └── AuthContext.tsx  # Контекст авторизации
│   │
│   ├── pages/               # Страницы приложения
│   │   ├── Login/           # Авторизация
│   │   │   ├── Login.tsx
│   │   │   └── Login.css
│   │   │
│   │   ├── Dashboard/       # Главная страница
│   │   │   ├── Dashboard.tsx
│   │   │   └── Dashboard.css
│   │   │
│   │   ├── Users/           # Пользователи
│   │   │   ├── Users.tsx
│   │   │   └── Users.css
│   │   │   ├── components/  # (для будущих компонентов)
│   │   │   ├── utils/       # (для будущих утилит)
│   │   │   └── hooks/       # (для будущих хуков)
│   │   │
│   │   ├── Admins/          # Администраторы
│   │   ├── Catalog/         # Каталог товаров
│   │   ├── Orders/          # Заказы
│   │   ├── Finance/         # Финансы
│   │   ├── Delivery/        # Доставка
│   │   ├── FAQ/             # FAQ
│   │   ├── Tickets/         # Тикеты поддержки
│   │   ├── Analytics/       # Аналитика
│   │   └── Database/        # Управление БД
│   │
│   ├── types/               # TypeScript типы
│   │   └── index.ts
│   │
│   └── utils/               # Утилиты
│       └── apiClient.ts     # HTTP клиент для API
│
├── public/                  # Статические файлы
└── package.json
```

## Принципы организации

### Каждая страница имеет свою структуру:

```
pages/
  Users/
    ├── Users.tsx           # Основной компонент страницы
    ├── Users.css           # Стили страницы
    ├── components/         # Компоненты, специфичные для этой страницы
    │   ├── UserList.tsx
    │   ├── UserForm.tsx
    │   └── UserDetail.tsx
    ├── utils/              # Утилиты для этой страницы
    │   └── userApi.ts
    └── hooks/              # Custom hooks (если нужны)
        └── useUsers.ts
```

## Роутинг

Роутинг настроен через `react-router-dom`:

- `/login` - Страница авторизации
- `/dashboard` - Главная страница
- `/users` - Управление пользователями
- `/admins` - Управление администраторами
- `/catalog` - Каталог товаров
- `/orders` - Заказы
- `/finance` - Финансы
- `/delivery` - Доставка
- `/faq` - FAQ
- `/tickets` - Тикеты поддержки
- `/analytics` - Аналитика
- `/database` - Управление БД

## Авторизация

Используется `AuthContext` для управления состоянием авторизации:
- Проверка токена через cookie
- Защищенные роуты через `ProtectedRoute`
- Автоматическое перенаправление на `/login` при отсутствии авторизации

## API клиент

HTTP клиент настроен в `src/utils/apiClient.ts`:
- Базовый URL из `REACT_APP_API_URL` или `http://localhost:8000`
- Автоматическая обработка ошибок (401 → redirect на login)
- Поддержка cookies для авторизации

## Следующие шаги

1. **Перенос функционала из старых HTML шаблонов:**
   - Изучить каждый HTML шаблон в `admin_web/templates/`
   - Создать соответствующие React компоненты
   - Реализовать логику работы с API

2. **Для каждой страницы нужно:**
   - Создать список элементов (если есть)
   - Создать форму создания/редактирования
   - Создать детальный просмотр (если есть)
   - Настроить API запросы в `utils/`
   - Добавить валидацию форм
   - Добавить обработку ошибок

3. **Компоненты для переноса:**
   - Users: список пользователей, детальный просмотр
   - Admins: список админов, создание/редактирование
   - Catalog: список товаров, форма создания/редактирования, детальный просмотр
   - Orders: список заказов, форма создания/редактирования
   - Finance: дашборд, транзакции, настройки, валютные переводы, продажи
   - Delivery: список доставок, статусы доставки
   - FAQ: список FAQ, форма создания/редактирования
   - Tickets: список тикетов, детальный просмотр
   - Analytics: графики и статистика
   - Database: управление таблицами БД

## Запуск

```bash
cd admin_react
npm install
npm start
```

## Сборка

```bash
npm run build
```

Собранное приложение будет в папке `build/`.
