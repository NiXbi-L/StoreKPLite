# Система уровней доступа админки

## Роли пользователей

В системе существует 4 уровня доступа:

### 1. **Owner (Владелец)**
- Полный доступ ко всем разделам
- Единственная роль, которая может управлять администраторами
- Единственная роль с доступом к прямому управлению базой данных
- Может создавать, редактировать и удалять администраторов других ролей

### 2. **Admin (Администратор)**
- Доступ к большинству разделов, кроме управления администраторами и базы данных
- Может управлять пользователями, каталогом, заказами, финансами, доставкой
- Имеет доступ к аналитике
- Не может управлять другими администраторами

### 3. **Moderator (Модератор)**
- Доступ к контенту и поддержке
- Может управлять пользователями, каталогом, заказами, доставкой
- Может работать с FAQ и тикетами
- Не имеет доступа к финансам, аналитике, управлению администраторами и базе данных

### 4. **Support (Поддержка)**
- Минимальный уровень доступа
- Доступ только к тикетам и FAQ
- Может просматривать дашборд
- Не имеет доступа к управлению пользователями, каталогом, заказам, финансам и другим разделам

## Описание разделов

### 📊 Дашборд
**Доступ:** Все роли  
**Описание:** Главная страница с общей статистикой системы

### 👥 Пользователи
**Доступ:** Owner, Admin, Moderator  
**Описание:** Управление пользователями системы - просмотр, редактирование, удаление

### 👤 Администраторы
**Доступ:** Только Owner  
**Описание:** Управление администраторами - создание, редактирование ролей, удаление

### 📦 Каталог
**Доступ:** Owner, Admin, Moderator  
**Описание:** Управление товарами и каталогом - добавление, редактирование, удаление товаров

### 🛒 Заказы
**Доступ:** Owner, Admin, Moderator  
**Описание:** Управление заказами - просмотр, изменение статусов, управление оплатой

### 💰 Финансы
**Доступ:** Owner, Admin  
**Описание:** Финансовые операции и отчеты - транзакции, счета, продажи, настройки финансов

### 🚚 Доставка
**Доступ:** Owner, Admin, Moderator  
**Описание:** Управление доставкой заказов - статусы доставки, редактирование информации

### ❓ FAQ
**Доступ:** Все роли  
**Описание:** Управление часто задаваемыми вопросами - создание, редактирование, удаление

### 🎫 Тикеты
**Доступ:** Все роли  
**Описание:** Обращения пользователей в поддержку - просмотр, ответы, изменение статусов

### 📈 Аналитика
**Доступ:** Owner, Admin  
**Описание:** Аналитика и статистика - графики, метрики, воронки конверсии

### 🗄️ База данных
**Доступ:** Только Owner  
**Описание:** Прямой доступ к базе данных - просмотр таблиц, редактирование записей, создание и удаление

## Техническая реализация

### Файлы системы прав доступа

1. **`src/utils/permissions.ts`**
   - Конфигурация уровней доступа
   - Функции проверки прав
   - Типы и интерфейсы

2. **`src/components/ProtectedSection.tsx`**
   - Компонент защиты разделов
   - Проверка прав доступа перед рендерингом
   - Отображение сообщения об отказе в доступе

3. **`src/components/Layout/Layout.tsx`**
   - Динамическое отображение меню на основе ролей
   - Показ только доступных разделов

4. **`src/App.tsx`**
   - Обертка всех роутов в `ProtectedSection`
   - Автоматическая проверка прав при переходе

### Использование в коде

```typescript
import { hasAccess, getCurrentRole, getAvailableSections } from '../utils/permissions';

// Проверка доступа к разделу
const canAccess = hasAccess(getCurrentRole(), 'finance');

// Получение списка доступных разделов
const sections = getAvailableSections(getCurrentRole());

// Защита компонента
<ProtectedSection sectionKey="finance">
  <Finance />
</ProtectedSection>
```

### Получение роли пользователя

Роль пользователя хранится в `localStorage` под ключом `admin_type` и устанавливается при авторизации через API `/users/admin/login`.

Возможные значения:
- `"owner"` - Владелец
- `"admin"` - Администратор
- `"moderator"` - Модератор
- `"support"` - Поддержка

## Безопасность

⚠️ **Важно:** Проверка прав на фронтенде является только UI-защитой. Все критичные операции должны дополнительно проверяться на бэкенде через соответствующие эндпоинты API.

Бэкенд проверяет права доступа через:
- `require_owner` - только для владельца
- `require_admin_or_owner` - для администратора или владельца
- `check_admin_access` - для всех администраторов (owner, admin)
